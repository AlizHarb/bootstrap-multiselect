Advanced Select Component with Search, Tags, Remote API & More

Bootstrap 5 Dark Mode Remote API

Overview

azh-select is a powerful, lightweight, and highly customizable select component built for Bootstrap 5. It provides advanced features like search, tags, remote API loading, custom templates, and full dark mode support.

Basic Template Example

Add icons and styling to your options:

const select = azhSelect.get('#iconSelect');
select.options.templateOption = (opt) => {
    const icon = document.querySelector(`option[value="${opt.value}"]`).dataset.icon;
    const color = document.querySelector(`option[value="${opt.value}"]`).dataset.color;
    return `<i class="bi ${icon} text-${color} me-2"></i>${opt.label}`;
};
select.options.templateValue = (opt) => {
    const color = document.querySelector(`option[value="${opt.value}"]`).dataset.color;
    return `<span class="badge bg-${color}">${opt.label}</span>`;
};

Template Best Practices

  • Keep it simple: Avoid overly complex HTML in templates
  • Use Bootstrap classes: Leverage Bootstrap's utility classes for consistent styling
  • Handle missing data: Always check if data attributes exist before using them
  • Escape HTML: If displaying user input, ensure it's properly escaped
  • Performance: Keep templates lightweight for large datasets
  • Accessibility: Ensure templates remain accessible with proper ARIA attributes
Safe Template Example
templateOption: (opt) => {
    // Safely get data attributes
    const option = document.querySelector(`option[value="${CSS.escape(opt.value)}"]`);
    const icon = option?.dataset.icon || 'bi-circle';
    const color = option?.dataset.color || 'secondary';

    // Escape HTML content
    const safeLabel = opt.label.replace(//g, '>');

    return `<i class="bi ${icon} text-${color} me-2"></i>${safeLabel}`;
}

azh-select triggers custom events that allow you to respond to user interactions and state changes.

Event List

Event Trigger Detail Object Description
azh:change Selection changes { instance } Fired when user selects/deselects options
azh:clear Clear button clicked { instance } Fired when clear button is clicked
azh:open Dropdown opens { instance } Fired when dropdown is opened

Basic Remote Setup

        data-remote="true"
        data-endpoint="https://api.example.com/users"
        data-search="true"

API Configuration

Attribute Description Example
data-endpoint API endpoint URL /api/users
data-value-field Field name for option values id, user_id
data-label-field Field name for option labels name, title, username
data-min-query Minimum characters to trigger search 2, 3
data-delay Debounce delay in milliseconds 300, 500
data-preload Load data on initialization true, false

Expected API Response Format

Your API should return a JSON array of objects:

[
    {
        "id": 1,
        "name": "John Doe",
        "email": "john@example.com",
        "role": "Admin"
    },
    {
        "id": 2,
        "name": "Jane Smith",
        "email": "jane@example.com",
        "role": "User"
    }
]
Field Mapping
  • data-value-field="id" ‚Üí Uses the id field for option values
  • data-label-field="name" ‚Üí Uses the name field for option labels
  • If fields don't exist, fallbacks are: id, name, title, then the value itself

Search Query Parameters

When search is enabled, queries are sent as URL parameters:

GET /api/users?q=john
Server-side Implementation Examples
PHP (Laravel)
Route::get('/api/users', function (Request $request) {
    $query = $request->get('q');

    return User::when($query, function ($q) use ($query) {
        return $q->where('name', 'LIKE', "%{$query}%")
                ->orWhere('email', 'LIKE', "%{$query}%");
    })
    ->select('id', 'name', 'email', 'role')
    ->limit(50)
    ->get();
});
Node.js (Express)
app.get('/api/users', (req, res) => {
    const query = req.query.q;

    let sql = 'SELECT id, name, email, role FROM users';
    let params = [];

    if (query) {
        sql += ' WHERE name LIKE ? OR email LIKE ?';
        params = [`%${query}%`, `%${query}%`];
    }

    sql += ' LIMIT 50';

    db.query(sql, params, (err, results) => {
        if (err) return res.status(500).json({ error: err.message });
        res.json(results);
    });
});
Python (Django)
from django.http import JsonResponse
from django.db.models import Q

def users_api(request):
    query = request.GET.get('q', '')

    users = User.objects.all()

    if query:
        users = users.filter(
            Q(name__icontains=query) |
            Q(email__icontains=query)
        )

    users = users.values('id', 'name', 'email', 'role')[:50]

    return JsonResponse(list(users), safe=False)

select._fetchRemote = function(query = '') {
    if (!this.options.endpoint) return;

Caching Results
// Simple cache implementation
const cache = new Map();

select._fetchRemote = function(query = '') {
    const cacheKey = `${this.options.endpoint}?q=${query}`;

    if (cache.has(cacheKey)) {
        // Use cached data
        const data = cache.get(cacheKey);
        this._processRemoteData(data, query);
        return;
    }

    // Fetch from API and cache result
    fetch(url)
        .then(res => res.json())
        .then(data => {
            cache.set(cacheKey, data);
            this._processRemoteData(data, query);
        });
};

Dynamic Theme Switching

Implement theme switching with JavaScript:

// Theme switcher function
function toggleTheme() {
    const html = document.documentElement;
    const currentTheme = html.getAttribute('data-bs-theme');
    const newTheme = currentTheme === 'dark' ? 'light' : 'dark';

    html.setAttribute('data-bs-theme', newTheme);

    // Optional: Save preference
    localStorage.setItem('theme', newTheme);
}

// Load saved theme on page load
document.addEventListener('DOMContentLoaded', () => {
    const savedTheme = localStorage.getItem('theme');
    if (savedTheme) {
        document.documentElement.setAttribute('data-bs-theme', savedTheme);
    }
});

        this.list.innerHTML = '';
        for (let i = startIndex; i < endIndex; i++) {
            const option = this.filteredOptions[i];

        }

Integration with Popular Libraries

React Integration
import React, { useEffect, useRef } from 'react';
import azhSelect from 'azh-select';

function AzhSelectComponent({ options, value, onChange, ...props }) {
    const selectRef = useRef();
    const instanceRef = useRef();

    useEffect(() => {
        // Initialize
        azhSelect.init(selectRef.current, props);
        instanceRef.current = azhSelect.get(selectRef.current);

        // Listen for changes
        selectRef.current.addEventListener('azh:change', onChange);

        // Cleanup
        return () => instanceRef.current?.destroy();
    }, []);

    useEffect(() => {
        // Update value
        if (instanceRef.current && value !== selectRef.current.value) {
            selectRef.current.value = value;
            instanceRef.current._render();
        }
    }, [value]);

    return (
        <select ref={selectRef} className="form-select azh-select">
            {options.map(opt =>
                <option key={opt.value} value={opt.value}>{opt.label}</option>
            )}
        </select>
    );
}

Troubleshooting

Common issues and solutions for azh-select implementation.

Common Issues

Select Not Initializing
Problem: azh-select doesn't transform the select element.
Solutions:
  • Ensure the element has azh-select class
  • Check that JavaScript is loaded after the DOM
  • Verify Bootstrap CSS is loaded
  • Check browser console for errors
// Correct initialization
document.addEventListener('DOMContentLoaded', function() {
    azhSelect.init('.azh-select');
});
Styling Issues
Problem: Select doesn't match your theme or looks broken.
Solutions:
  • Ensure Bootstrap 5 CSS is loaded before azh-select CSS
  • Check CSS custom properties are supported
  • Verify data-bs-theme attribute is set correctly
  • Override CSS variables for custom styling
Remote API Not Working
Problem: Remote data doesn't load or search doesn't work.
Solutions:
  • Check API endpoint is accessible
  • Verify CORS headers are set on server
  • Ensure API returns JSON array format
  • Check field mapping with data-value-field and data-label-field
// Debug remote issues
document.querySelector('#remoteSelect').addEventListener('azh:error', function(e) {
    console.error('API Error:', e.detail.error);
});
Events Not Firing
Problem: Custom event listeners don't trigger.
Solutions:
  • Add event listeners to the original select element
  • Use correct event names (prefixed with azh:)
  • Ensure listeners are added after initialization
// Correct event listener
const select = document.querySelector('#mySelect');
azhSelect.init(select);
select.addEventListener('azh:change', function(e) {
    console.log('Changed!');
});
